Audit and repair this repo as a hybrid Farcaster Mini App + website.

Goal:
- Make the app work inside Farcaster Mini Apps and as a normal website.
- Fix non-working buttons.
- Fix Farcaster connection/auth.
- Implement wallet connection for the wallet quest.
- Make quest actions and verification match each quest type.
- Keep Farcaster quest verification via Neynar.
- Do not fake X verification.

Constraints:
- Use full-file replacements when needed.
- Do not keep the legacy /api/auth/session flow.
- Do not use window.open for Mini App navigation.
- Do not render one generic Verify button for every quest.
- Do not claim X quests are verified unless X API support is actually added.

Required doc-aligned behavior:
- Use sdk.isInMiniApp() to branch Mini App behavior vs regular web behavior.
- In Mini App mode, use sdk.quickAuth.fetch for authenticated requests.
- Validate Quick Auth on the server before creating a session.
- Use sdk.actions.ready() after initialization.
- Use sdk.actions.openUrl for external URLs.
- Use sdk.actions.viewProfile for Farcaster follow quests.
- Use sdk.actions.viewCast for Farcaster cast/recast quests.
- Use Wagmi with @farcaster/miniapp-wagmi-connector for wallet connection in Mini App mode.

Install:
npm install @farcaster/miniapp-wagmi-connector jose

- Inside Farcaster Mini App:
  - Connect with Farcaster works
